# 当前开发进度与交接

这份文档用于记录今天已经完成的工作、当前真实可用能力，以及接下来最推荐继续推进的任务，方便在后续窗口或下一次开发时快速接上。

## 1. 今天已完成的内容

### 1.1 后端基础规范层已落地

已完成：

- 统一返回体 `ApiResponse`
- 全局异常处理 `GlobalExceptionHandler`
- 基础分页请求/响应对象
- MyBatis-Plus 分页能力接入

说明：

- 当前后端接口已经统一使用 `ApiResponse`
- 前端请求层已经完成对 `ApiResponse` 的统一适配

### 1.2 登录注册与 JWT 认证已打通

已完成：

- `POST /api/auth/login`
- `POST /api/auth/register`
- `GET /api/auth/me`

已接入能力：

- Spring Security
- JWT 鉴权
- 登录态恢复
- 前端请求自动携带 `Authorization: Bearer ...`

说明：

- 前端登录/注册已经接到真实后端接口
- 页面刷新后，前端会通过 `/api/auth/me` 恢复登录态
- 用户角色模型已加入后端，目前至少支持 `USER` 与 `ADMIN`

### 1.3 trails 公共列表接口已落地

已完成：

- `GET /api/trails`
- `GET /api/trails/{id}`

`GET /api/trails` 当前已支持：

- `keyword`
- `difficulty`
- `packType`
- `durationType`
- `distance`
- `sort`
- `pageNum`
- `pageSize`

当前排序语义：

- `latest`
- `hot`
- `rating`

说明：

- 后端已经采用“一个公共 trails 列表接口”的策略
- 没有拆分首页、搜索页、社区页专用接口

### 1.4 前端列表页已改为真实接口

已完成真实接口对接的页面：

- 首页
- 搜索页
- 社区页

当前策略：

- 首页调用 `GET /api/trails?sort=hot&pageNum=1&pageSize=6`
- 搜索页调用 `GET /api/trails` 并把搜索词与筛选条件交给后端
- 社区页调用 `GET /api/trails?sort=latest`

已完成配套：

- 前端统一 trails API 层
- 页面级数据适配层
- 加载态
- 空状态
- 错误态

### 1.5 路线详情主数据已接入真实后端

已完成：

- `TrailDetailView` 主体数据已接入 `GET /api/trails/{id}`

当前详情页真实化范围：

- 路线基础信息
- 作者信息
- 难度/距离/时长等展示
- 地图定位与天气联动所需的 `ip`、`location`

目前已经能跑通的真实链路包括：

- 用户注册

## 5. 最推荐的下一步

最推荐马上开始的任务是：

1. 评论接口后端化
2. 详情页评论区接真实接口

推荐原因：

- 详情页主数据已经完成真实化
- 现在最明显的缺口就是评论区还在 mock
- 只要评论接完，“列表页 -> 详情页 -> 评论互动”主链路就会闭环

### 推荐先做的接口

优先建议：

- `GET /api/trails/{id}/reviews`
- `POST /api/reviews`

其中评论模块需要明确支持：

- 顶级评论
- 二级回复
- 三级回复
- 二级/三级图片回复
