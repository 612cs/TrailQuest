# DOCS 索引

`DOCS` 目录按用途分为四类，方便在继续开发时快速定位资料和阶段状态。

## 推荐阅读顺序

1. `handoff`：先看最新交接文档，了解当前进度与下一步
2. `backend`：再看后端阶段总结与开发清单
3. `database`：需要落库、改表、导数据时看数据库资料
4. `frontend`：涉及 UI 规范时查看前端文档

## 分类说明

### `handoff`

- 阶段交接与每日状态总结
- 建议每次阶段性收工后新增一份 dated handoff

文档：

- `handoff/current-progress-handoff-2026-03-15.md`
- `handoff/current-progress-handoff-2026-03-16.md`
