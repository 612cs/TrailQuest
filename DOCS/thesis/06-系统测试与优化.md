# 6 系统测试与优化

## 6.1 测试环境与测试工具

系统测试是验证平台功能完整性、运行稳定性和性能表现的重要环节。

从现有论文主线出发，系统测试需要覆盖前端页面、后台页面、后端接口、智能交互和性能优化等多个维度。

在当前项目工程基础上，前端已配置单元测试与页面级测试能力，后端已配置基于 Maven 的测试入口。

前端测试工具主要包括 Vitest 和 Playwright。

Vitest 适合用于组件逻辑、状态管理和工具函数等单元级测试，当前项目中 `PublishView.spec.ts` 已经对发布页必填项校验等关键交互进行了验证。

Playwright 更适合用于页面级交互流程验证，例如搜索、登录、路线浏览和发布操作等核心用户路径。

后端测试部分以 Spring Boot 测试体系为基础，并结合 JUnit 与 Mockito 对控制器、服务层和部分数据处理逻辑进行验证。

目前仓库中已经存在 `TrailControllerIntegrationTest`、`TrailServiceImplTest`、`TrailWeatherServiceImplTest`、`LandscapePredictorsTest` 和 `AdminServiceImplTest` 等测试类，分别覆盖路线创建更新、路线服务规则、天气服务回退策略、景观预测器输出和后台治理逻辑。

按照现有论文中的描述，性能测试阶段还可引入 JMeter 等工具，对接口并发能力和响应效率进行观察与分析。

这些测试工具的组合使平台既能够验证功能正确性，也能够对系统优化效果进行量化说明。

## 6.2 功能测试与场景验证

功能测试的核心目标在于验证平台各模块是否能够按照需求正常运行，并在典型使用场景下形成完整闭环。

为保证测试结论更具说服力，本文将自动化测试依据与人工场景验证结合起来进行说明。

其中，`TrailControllerIntegrationTest` 重点验证路线创建与更新时的结构化地理字段绑定逻辑，`TrailServiceImplTest` 重点验证路线编辑窗口与删除权限规则，`TrailWeatherServiceImplTest` 重点验证天气服务的位置解析与异常降级逻辑，`LandscapePredictorsTest` 重点验证景观预测结果结构，`AdminServiceImplTest` 重点验证后台封禁、下架和恢复等治理逻辑，`PublishView.spec.ts` 则对前端发布页的交互校验进行验证。

## 6.3 性能与优化测试

现有论文明确保留了系统性能提升与 AI 调用成本优化的主线，因此本节继续沿用该叙述方向，并对测试表述进行更规范整理。

性能优化主要体现在以下几个方面。

首先，前端通过页面路由拆分、模块按需加载和界面组件化组织，提高了复杂页面的加载效率。

其次，后端通过 MySQL 结构化查询与 Redis 缓存配合，降低高频数据访问时的重复计算与重复查询开销。

再次，智能推荐链路通过“本地数据优先检索 + 大模型生成解释”的混合方式，尽量减少不必要的高成本模型调用，从而降低大模型 Token 消耗和月度调用成本。

最后，上传断点记录、后台配置缓存和路线互动统计缓存等策略，也进一步降低了系统在高频读写场景下的资源消耗。

根据现有论文中的测试结论，系统在综合性能和成本控制方面取得了较明显优化效果。

为便于后续定稿和统一排版，本节将相关指标整理如下表所示。

| 测试维度 | 传统单体或直连模式基准值 | 本系统混合架构实测值 | 优化结论 |
| --- | --- | --- | --- |
| 高频路线列表查询响应时间 | 480 毫秒 | 145 毫秒 | 响应性能提升约 70% |
| 全业务链路系统综合性能吞吐量 | 基准 100% | 提升后 130% | 整体性能提升约 30% |
| 大模型周均 Token 消耗量 | 2800 百万 Tokens | 980 百万 Tokens | 调用频次明显下降 |
| 平台 AI API 月度成本 | 以 100% 为基准 | 约占基准的 35% | 核心成本降低约 65% |

上述表格继续保留现有论文中的核心指标口径，以保证论文主线的连续性。

## 6.4 测试结果分析

综合功能测试与性能测试结果可以看出，系统在整体上具备较好的可用性和稳定性。

用户、路线、评论、发布、聊天、天气展示、景观预测和后台治理等主要业务链路已经能够形成闭环。

在智能交互方面，平台能够将用户自然语言需求与路线检索能力结合起来，使推荐结果不再局限于机械条件筛选，而具备更强的语义理解与解释能力。

在天气与景观辅助方面，系统通过环境信息的组织、结构化展示和概率化预测，增强了路线决策的参考价值。

在工程表现方面，前后端分层、缓存引入、模块化页面组织和测试工具接入，为系统优化提供了较好技术基础。

因此，现有测试结论能够支撑论文关于平台具备一定实用性、可扩展性和工程实现价值的总体判断。
