# 2 相关技术概述

## 2.1 前端技术栈

本系统前端部分以 Vue 为核心开发框架，结合 TypeScript、Vite、Tailwind CSS、DaisyUI、Pinia 与 Vue Router 等技术完成页面构建、状态管理、路由控制与界面风格统一。

Vue 具有组件化开发能力强、响应式机制完善和生态成熟等优点，适合用于构建交互复杂、界面层次较多的单页应用。

在本项目中，首页、搜索页、社区页、个人中心、发布页、聊天页和路线详情页等功能页面均可以通过组件化方式进行组织，从而提高代码复用度和维护效率。

Vue 3 的响应式系统在处理表单输入、筛选条件变化、对话消息刷新和地图区域联动时具有较好的表现。

TypeScript 为前端工程提供了静态类型约束。

在系统中，路线对象、评论对象、用户信息、天气数据和 AI 对话数据都具有较明确的数据结构。

通过 TypeScript 对接口、数据对象和组件参数进行约束，可以降低运行阶段因字段不一致而造成的问题，提高前后端协同开发的稳定性。

Vite 作为现代前端构建工具，具备启动速度快、热更新高效和构建配置简洁等特点。

在当前项目工程中，前端代码组织于 `apps/web` 目录，开发命令、构建命令和测试命令均已围绕该目录进行配置。

这使系统在持续迭代过程中能够保持较高的工程效率。

Tailwind CSS 与 DaisyUI 共同承担界面样式层的构建任务。

Tailwind CSS 通过功能类方式提高页面搭建效率，便于快速表达卡片、布局、颜色、间距和响应式行为。

DaisyUI 则有助于统一按钮、表单、弹窗和列表等常见界面元素的设计风格，使系统在整体视觉体验上更加协调。

此外，系统在前端页面中还引入了 Pinia 与 Vue Router。

Pinia 负责全局状态管理，适合处理登录信息、用户资料、路线交互状态和 AI 会话状态等跨组件共享数据。

Vue Router 则负责页面路由配置和视图切换，是实现首页、搜索页、路线详情页和聊天页等页面联动的基础能力。


## 2.2 后端与服务框架

系统后端以 Spring Boot 3 为核心框架，配合 MyBatis-Plus、Spring Security、JWT、MySQL 和 Redis 完成业务逻辑处理、数据访问、安全认证和缓存支持。

Spring Boot 3 具有配置简洁、生态成熟和开发效率高的特点，适合构建中小型到中大型 Web 服务系统。

本课题中的用户管理、路线管理、评论管理、上传管理、配置管理和 AI 相关接口，都可以在 Spring Boot 提供的基础能力上完成组织与实现。

当前项目后端代码位于 `apps/api` 目录，并采用模块化方式组织控制器、服务层、数据访问层和数据传输对象。

MyBatis-Plus 作为数据访问框架，能够在保留 SQL 灵活性的同时，减少基础 CRUD 代码编写量。

由于本系统涉及用户、路线、评论、收藏、对话会话、媒体资源和轨迹数据等多类业务对象，采用 MyBatis-Plus 有利于提升开发效率并保持数据访问结构清晰。

MySQL 是系统的核心持久化数据库。

用户资料、路线信息、社区评论、媒体文件和 AI 会话记录等核心业务数据均需要稳定存储和结构化管理，因此 MySQL 在本系统中承担基础数据存储任务。

Redis 用于提升高频访问场景下的响应效率。

系统在热点数据缓存、高频操作状态维护和部分短期数据存储方面引入 Redis，有助于减少数据库的直接压力，提升整体访问性能。

在安全机制方面，系统采用 Spring Security 与 JWT 结合的方式完成身份认证和访问控制。

用户登录成功后由服务端签发令牌，前端在后续请求中携带令牌访问受保护资源。

这种方式既能满足前后端分离架构下的认证需求，也为用户中心、评论、发布与管理等功能提供了必要的安全基础。



## 2.3 人工智能与 GIS 技术

本系统的智能能力主要建立在大语言模型、地理信息服务和景观预测模型之上。

在智能交互层面，系统设计中引入 DeepSeek 大语言模型，用于理解用户的自然语言输入，并识别其中包含的地点、时间、难度、偏好景观和活动意图等关键信息。

相较于传统基于关键词的检索方式，大语言模型能够更好地处理模糊、组合式和口语化表达，从而使系统更接近真实使用场景中的交互习惯。

为了使大语言模型能够与底层业务能力进行有效联动，系统在技术路线中引入 Function Calling 机制。

该机制的核心作用是将模型理解出的意图和参数，组织成系统能够执行的结构化调用请求。

通过这一方式，平台可以在自然语言理解、路线检索、条件筛选和推荐结果生成之间形成闭环。

GIS 能力是本系统区别于一般内容平台的重要技术基础。

项目通过高德地图 API 实现地图展示、地理定位、路线可视化和空间信息呈现。

路线详情页中的起终点展示、轨迹预览、位置提示以及相关空间信息说明，都需要依赖地图服务能力完成。

在当前项目中，前端已经形成与地图相关的页面组织方式，后端也围绕地理与天气数据组织了辅助接口，这为论文中相关设计与实现内容提供了补充依据。

在环境感知与景观辅助决策方面，系统按照既有材料中的技术路线，引入 Python Flask 封装的随机森林预测模型。

随机森林适合处理多因素共同作用下的分类与概率预测问题，能够将时间、天气、地理环境等数据融合起来，对云海、日出等特定景观的出现概率进行估计。

这一能力使平台不仅具备路线推荐功能，也具备一定的景观机会识别和出行辅助判断能力。

综合来看，前端技术栈保证了系统的交互体验与可视化表现，后端框架提供了稳定的业务支撑与安全机制，AI 与 GIS 技术则构成了平台的智能服务核心。

这些技术共同支撑了户外路线智能推荐平台的整体实现。
