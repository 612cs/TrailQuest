基于Vue的户外路线智能推荐平台的设计与实现

答辩人：陈胜 学号：202201150626 专业：软件工程 指导教师：昌明权 刘妙兴
老师们好，我是软件工程专业的陈胜。今天汇报的题目是“基于Vue的户外路线智能推荐平台的设计与实现”。这套系统围绕户外徒步路线发现、地图轨迹展示、AI智能推荐、天气与景观预测以及后台审核治理展开，目标是把用户找路线、判断路线、发布路线和平台管理这些环节做成一个相对完整的闭环。

第1章 绪论

Research Background

户外路线需求从“能找到”走向“适合我”

普通用户开始关注路线难度、天气窗口、景观概率和自身能力匹配，传统热度排序已经难以满足个性化决策。

轨迹数据不完整路线介绍和真实轨迹割裂，用户难以判断风险。
环境信息不及时天气、景观窗口和安全提醒缺少联动。
推荐解释不充分按热度或关键词筛选，缺少自然语言理解。
近年来户外徒步参与人群明显扩大，用户已经不只满足于看到一条路线名称，而是希望知道这条路线是否适合自己的体力、什么时候去更合适、能否看到云海日出等景观。现有工具往往分散在轨迹、天气、社区内容和路线搜索之间，缺少针对中文户外场景的一体化推荐平台。所以本文希望把路线数据、地图轨迹、天气环境和AI对话结合起来，提升路线选择的效率和可信度。

第1章 绪论

国内外研究现状

国外研究

  • 路线规划从最短路径转向质量感知与多目标优化。
  • 大模型工具调用为自然语言推荐提供新交互范式。
  • 随机森林等机器学习方法用于能见度与雾类天气预测。

国内研究

  • 智慧文旅平台建设较成熟，Spring Boot + Vue 实践丰富。
  • 地图 API、前后端分离和后台治理已有较多工程经验。
  • 智能推荐开始关注透明性、用户行为和可解释性。
研究空缺：面向户外徒步垂直场景，将 GIS 地图、天气环境、用户画像、AI 对话和景观预测整合的系统仍然较少。
从文献看，国外研究更关注质量感知导航、多目标路线优化、大模型工具调用和气象预测模型；国内研究更重视智慧文旅平台建设、地图服务接入以及前后端分离工程实践。本文的切入点不是单独提出一个算法，而是把这些能力整合到中文户外徒步这个具体场景中，让用户可以通过自然语言表达需求，再由系统完成路线检索、评分、解释和环境辅助判断。

第2章 技术概述

相关技术概述

前端Vue 3

Composition API、TypeScript、Vite、Pinia、Tailwind CSS、DaisyUI

后端Spring Boot 3

MyBatis-Plus、MySQL、Redis、JWT、Spring Security

AI 与 GISDeepSeek + 高德地图

Function Calling、路线语义解析、地图轨迹渲染、随机森林景观预测

系统采用前后端分离结构。前端以 Vue 3 和 TypeScript 为主，负责页面交互、地图展示和状态管理；后端使用 Spring Boot 3、MyBatis-Plus、MySQL 和 Redis，负责用户、路线、评论、媒体和审核等业务；AI与GIS部分使用 DeepSeek 做自然语言理解和推荐理由生成，地图服务负责轨迹展示，随机森林模型用于云海、雾凇、冰挂、日出日落等景观概率预测。

第3章 需求分析

系统功能需求

普通用户端

注册登录路线浏览搜索路线详情地图路线发布上传社区互动个人中心画像AI智能推荐天气景观预测

管理员端

用户管理路线审核路线管理评论治理系统配置操作日志
功能需求分为普通用户端和管理员端。普通用户需要完成找路线、看地图、发布路线、参与评论、维护个人画像以及使用AI推荐和天气景观预测。管理员端则负责用户管理、路线审核、路线治理、评论治理、配置和日志。这样的划分可以保证前台体验和后台治理同时闭环，也符合毕业设计对完整系统的要求。

第4章 总体设计

系统架构设计

  • 前端展示层：Vue 3、Pinia、高德地图 API。
  • 后端服务层：鉴权、业务逻辑、数据持久化、Redis 状态。
  • AI服务层：语义解析、Function Calling、景观预测。
系统整体采用三层结构。前端展示层负责用户交互、地图可视化和页面渲染；后端服务层负责鉴权、业务规则、数据库和短期状态管理；AI服务层负责自然语言理解、推荐参数提取、推荐话术生成以及随机森林景观预测。这样的分层使系统既能保持工程结构清晰，也便于后续替换模型或扩展新的智能服务。
湖南信息学院
第4章 总体设计

系统功能模块

  • 前台功能覆盖路线、社区、用户画像与智能服务。
  • 后台功能覆盖审核、治理、权限与操作日志。
  • 大模型模块负责语义理解、工具调用和推荐解释。
功能模块围绕三个板块展开。第一是前台用户功能，包括路线浏览、路线发布、评论互动和个人中心；第二是后台管理员功能，包括审核、治理、配置和日志；第三是大模型智能功能，负责把用户的模糊自然语言需求转换为结构化路线检索条件，并把推荐结果以可解释方式反馈给用户。

第4章 核心链路

AI推荐与景观预测链路

自然语言输入语义解析路线检索评分景观概率预测推荐理由输出
这一页是系统的核心创新链路。用户用自然语言描述出行需求后，大模型提取地点、时间、难度和偏好等结构化参数，后端路线库完成候选检索和综合评分，最后输出路线卡片和推荐理由。景观预测链路则把路线坐标、海拔、天气和天文数据转成特征，交给随机森林预测器输出景观概率、风险等级和出行建议。

第5章 系统实现

路线详情与地图展示

基础信息、轨迹、天气、景观预测与评论互动在同一详情页聚合；地图使用高德 API 渲染路线折线、起终点与边界区域。
路线详情页是用户决策的主要入口。页面聚合路线名称、位置、难度、距离、爬升、封面介绍、地图轨迹、天气景观和评论互动。轨迹部分支持对 GPX、KML、GeoJSON 等数据进行解析和渲染，通过地图折线展示路线走向，并标注起终点，帮助用户更直观地判断路线特点。

第5章 系统实现

AI智能推荐实现

可解释推荐

自然语言需求先解析为结构化条件，再完成路线检索、评分排序和推荐理由生成。标准检索走数据库，模糊需求才调用 AI，兼顾体验与成本。

AI智能推荐模块允许用户直接用自然语言表达需求，例如“周末想找一条适合新手、能看日出的路线”。系统会先解析出地点、时间、难度和偏好，再调用后端路线检索和评分逻辑，最后以路线卡片和推荐理由的形式返回。这样推荐结果不是黑盒输出，而是能解释为什么推荐这条路线。
湖南信息学院
第5章 系统实现

天气与景观预测

环境辅助判断

输入路线坐标、轨迹海拔、天气与天文数据，输出云海、雾凇、冰挂、日出日落概率，以及风险等级和出行建议。

天气与景观预测模块用于补充路线选择中的环境因素。系统根据路线坐标、轨迹海拔和天气数据构造特征，再使用随机森林模型预测云海、雾凇、冰挂、日出日落等景观概率。随机森林训练成本较低、解释性较强，适合在毕业设计阶段做小样本原型验证。

第5章 系统实现

后台管理实现

审核链路：用户提交 → 管理员审核 → 通过或驳回；结合 RBAC 权限控制与操作日志审计。
后台管理模块主要保证平台内容治理和运营安全。管理员可以查看总览数据，对用户提交的路线进行审核，也可以进行路线管理、评论治理、配置维护和日志查看。路线审核支持通过和驳回，并保留操作记录，配合角色权限控制，可以满足平台基本治理需求。

第7章 总结

总结与展望

工作总结

  • 实现面向户外徒步场景的路线智能推荐平台。
  • 完成 Vue 3 前端、Spring Boot 后端与 AI 服务集成。
  • 形成“人找路线为主，条件化推荐为辅”的服务模式。

未来展望

  • 引入多模态路线画像，提取图片与视频中的地貌、路况特征。
  • 积累更多实测样本后，引入时间序列或空间特征模型。
总体来看，本文完成了一个基于 Vue 3 和 Spring Boot 的户外路线智能推荐平台，并集成 DeepSeek 大模型和随机森林景观预测能力。系统能够覆盖路线发现、发布、详情地图、社区互动、智能推荐、天气景观预测和后台管理。后续可以继续扩展多模态路线画像，并在积累更多实测数据后升级景观预测模型。
