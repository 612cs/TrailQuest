### 第3页 — 研究背景（第1章）

**核心要点：**

- 户外运动参与人群爆发式增长，从专业玩家扩大到普通大众
- 用户需求升级：不再满足简单路线介绍，关注路线是否适合自身水平、天气是否合适、能否看到独特风景
- 现有工具的三大痛点：
  1. 轨迹数据不完整
  2. 天气信息更新不及时
  3. 推荐功能大多按热度排序，缺乏个性化
- 一句话收束：需要一个专门面向户外徒步场景的智能推荐平台

---

### 第4页 — 国内外研究现状（第1章）

**国外研究：**
- 路线规划从最短路径 → 质量感知、多目标优化
- 大模型工具调用（ToolFlow、Meta-Tool）提供新的交互范式
- 随机森林等ML方法用于气象预测

**国内研究：**
- 智慧文旅平台建设成熟（Spring Boot + Vue 技术方案）
- 地图API整合、前后端分离工程实践丰富
- 推荐算法应用开始关注透明性和用户行为

**现状综述（核心论点）：**
- 现有研究分散在导航、旅游、AI、气象等领域
- 缺少面向中文户外徒步场景，将GIS地图 + 天气 + 用户画像 + AI对话 + 景观预测整合在一起的系统

### 第9页 — AI推荐与景观预测链路（第4章-3）

**这是论文的创新核心，需要清晰展示：**

**AI推荐链路：**
> 用户自然语言输入 → DeepSeek语义解析 → 结构化参数提取（地点/时间/难度/偏好） → 后端路线库检索 → 综合评分排序 → 推荐路线卡片 + 推荐理由

**综合评分公式（可选展示）：**
- 基础匹配 + 用户偏好 + 互动热度 + 环境景观 = 加权综合得分

**景观预测链路：**
> 路线坐标/海拔 + 天气/天文数据 → 特征构造（温湿度/风速/云量/降水/露点差） → 随机森林预测器 → 景观概率 + 风险等级 + 出行提示
