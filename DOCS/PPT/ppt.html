01 研究背景
  • 受众群体爆发：户外运动从少数专业玩家扩大到普通大众。
  • 用户需求升级：不再满足简单图文，更关注路线适配度、气象窗口及独特自然景观。
  • 现有工具三大痛点：
    1. 轨迹数据不完整，解析困难。
    2. 局部气象信息更新滞后。
    3. 推荐逻辑死板，多按热度排序，缺乏个性化。

结论：迫切需要一个专门面向户外徒步场景的智能推荐平台。

01 国内外研究现状

国外研究现状

  • 路线规划从最短路径转向质量感知与多目标优化。
  • 大模型工具调用架构(ToolFlow等)提供全新交互范式。
  • 随机森林等机器学习方法应用于微地形气象预测。

国内研究现状

  • 智慧文旅平台常采用 Spring Boot + Vue 架构。
  • GIS地图API整合与前后端分离实践经验丰富。
  • 推荐算法应用开始重点关注透明性和真实行为反馈。

核心论点 (现状综述)

现有研究多分散在独立领域。本项目旨在填补空白，打造一个将 GIS地图 + 天气数据 + 用户画像 + AI对话 + 景观预测 深度整合的中文专属系统。

03 系统功能需求分析

普通用户端 (C端)

  • 核心业务： 路线浏览、搜索、详情与GIS地图查阅。
  • 内容贡献： 户外路线发布、轨迹上传、社区互动。
  • 个性服务： 注册登录、个人中心、体能画像设置。
  • 智能赋能： AI自然语言对话推荐、气象景观预测。

系统管理员端 (B端)

  • 用户管理： 信息维护、角色分配与账号状态。
  • 路线管理： 全生命周期管理、人工合规审核与驳回。
  • 社区治理： 违规评论清理、精选优质路线发布。
  • 系统运维： 全局参数配置、操作日志审计监控。

05 系统实现：路线详情与地图展示
  • 数据聚合展示：页面顶部直观展示封面、难度、总距离、累计爬升等硬核基础信息。
  • GIS可视化：基于高德地图API，动态渲染GeoJSON(GPX/KML)轨迹折线，标注起终点。
  • 一站式服务：页面下方无缝集成区域天气趋势、景观预测面板及驴友实地评论。
05 系统实现：AI 智能对话推荐
  • 交互范式转变：从传统的"表单检索"变为类微信的"自然语言对话"。
  • 流程机制：大模型解析意图 → Function Calling结构化 → 本地数据库检索 → 卡片渲染。
  • 防幻觉设计：推荐结果具备可解释性，刚性检索走本地库，模糊长尾需求才依赖大模型推理，控制成本。
05 系统实现：天气与景观预测
  • 数据输入：提取路线核心坐标与平均海拔，结合第三方气象/天文API。
  • 模型应用：针对"云海"、"雾凇"、"日出"等，利用随机森林算法分别训练预测模型。
  • 直观输出：生成预测日历，展示发生概率百分比及出行气象风险等级。
  • 优势：训练成本低，特征重要性解释性强，极适合原型验证阶段。
05 系统实现：后台管理
  • 数据总览 Dashboard：可视化展示新增用户、发布趋势等运营数据。
  • 内容审核闭环：用户提交 → 待审队列 → 管理员核查地图/信息 → 通过或填写理由驳回。
  • 权限与审计：基于 Spring Security 的 RBAC 角色权限控制，并记录关键操作日志审计。
06 系统测试结论
  • 功能覆盖：全面覆盖用户认证、路线服务、社区互动、AI推荐、后台管理等核心场景。
  • 链路闭环：主要业务流（如路线上传审核、AI检索交互）均已打通，接口状态稳定。
  • 性能优化：实施了路由按需加载、地图延迟加载、列表虚拟滚动以及 SSE 流式响应，显著提升体验。
  • 最终结论：系统功能完整性与运行稳定性满足毕业设计既定目标，具备部署运行的原型基础。
07 总结与展望

阶段工作总结

  • 实现了一个聚焦于户外垂直场景的智能推荐平台。
  • 深度整合 Vue3 + SpringBoot + DeepSeek + 随机森林 交叉技术。
  • 探索形成了"以人找路线为主，多条件AI推荐为辅"的新服务模式。
  • 核心架构稳健，满足毕设检验要求。

未来改进方向

  • 多模态路线画像增强：尝试引入计算机视觉(CV)，从用户上传的实景图中自动提取地貌特征与实时路况信息。
  • 景观预测引擎升级：待积累海量样本后，引入深度时间序列模型(如LSTM)或空间联合分析，进一步提升气象预测精准度。
