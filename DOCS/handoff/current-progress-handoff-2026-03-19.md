# 当前开发进度与交接

这份文档用于记录 2026-03-19 已完成的工作、当前真实可用能力、剩余未完成项，以及下一步最推荐继续推进的方向，方便下一次开发时直接接上。

## 1. 今天已完成的内容

### 1.1 用户资料真实化已完成

已完成：

- `users` 表补齐真实 `bio` / `location` 字段文档
- 个人资料编辑页已接真实保存接口
- `/api/auth/me`、登录返回、注册返回已返回真实资料字段
- 顶栏头像、个人主页、用户资料恢复链路已接真实数据

说明：

- 当前 `bio` / `location` 不再依赖前端硬编码兜底
- 当前用户资料刷新后不会再退回到 mock/default 文案

### 1.2 头像编辑已改为“上传后保存生效”

已完成：

- 个人主页头像可点击更换
- 上传完成后先拿到 `mediaId`
- 点击“保存更改”后才真正更新 `avatar_media_id`
- 取消编辑不会改动正式头像

说明：

- 现在不再是“上传即修改用户头像”
- 头像编辑与基础资料保存的交互已经统一

### 1.3 注册补充信息页已完成

已完成：

- `AuthModal` 从单页登录/注册改为三态流程
- 点击“立即注册”后先进入补充信息页
- 用户可填写徒步画像，也可以直接跳过
- 跳过后进入注册页，正常注册
- 填写后进入注册页，注册时一并提交补充信息

当前注册三步态：

- 登录页
- 注册前补充信息页
- 注册账号页

说明：

- 当前不新增独立注册路由
- 整个流程仍在现有认证弹窗体系内完成

### 1.4 徒步画像建模与个人主页入口已完成

已完成：

- 新增 `user_hiking_profiles` 一对一画像表设计与代码接入
- 画像字段包括：
  - `experience_level`
  - `trail_style`
  - `pack_preference`
- “地区”继续复用 `users.location`
- 个人主页新增“徒步画像”入口
- 个人主页可打开独立画像编辑弹窗

当前画像字段第一版：

- 经验等级：`beginner / intermediate / expert`
- 常走类型：`city_weekend / long_distance / both`
- 负重偏好：`light / heavy / both`

说明：

- 画像数据已走真实落库，不是前端暂存
- 注册成功后、刷新后、进入个人主页后都能拿到当前画像数据

### 1.5 当前用户返回结构已扩展

已完成：

- `CurrentUserVo` 增加 `hikingProfile`
- 前端 `CurrentUser` / `UserProfile` 已同步扩展
- `PUT /api/users/me/hiking-profile` 已新增

说明：

- 当前基础资料编辑与画像编辑分为两条接口
- 基础资料仍走 `PUT /api/users/me/profile`
- 徒步画像单独走 `PUT /api/users/me/hiking-profile`

### 1.6 徒步画像枚举映射问题已修复

已完成：

- 修复 `user_hiking_profiles` 查询时的枚举映射异常
- 保持数据库继续存小写 code 值
- 后端枚举改为按值映射，而不是按枚举常量名映射

本次修复覆盖：

- `HikingExperienceLevel`
- `HikingTrailStyle`
- `HikingPackPreference`

说明：

- 之前报错根因是数据库存的是 `intermediate`，MyBatis 默认按 `INTERMEDIATE` 读取
- 当前这条链路已经可以直接读取数据库中的小写值

## 5. 下一步最推荐的工作

最推荐优先推进的任务是：

1. 登录失败时补“用户不存在 -> 引导注册”
2. 社区平台路线卡片分享按钮接真实分享能力
3. 规划并落地 GPX / KML 轨迹文件上传与解析

推荐原因：

- 当前注册补充信息页已经做好，登录失败时引导注册是最自然的体验补齐
- 社区卡片分享按钮属于现有页面上肉眼可见的能力缺口
- GPX / KML 仍然是后续路线内容能力的基础建模，但优先级已经低于当前 auth / 社区体验断点

### 建议优先级判断

最值得优先补的是：

1. 登录引导注册
2. 社区卡片分享
3. GPX / KML 方案

说明：

- 前两项都属于已存在功能链路上的真实缺口
- GPX / KML 虽然重要，但更偏中等规模建模任务
