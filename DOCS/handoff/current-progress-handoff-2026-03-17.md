# 当前开发进度与交接

这份文档用于记录 2026-03-17 已完成的工作、当前真实可用能力、剩余未完成项，以及下一步最推荐继续推进的方向，方便下一次开发时直接接上。

## 1. 今天已完成的内容

### 1.1 路线详情互动闭环已完成

已完成：

- 详情页分享能力接入
- 点赞 / 取消点赞真实接口接入
- 收藏 / 取消收藏真实接口接入
- 首页、搜索页、社区页、详情页的互动状态联动
- 未登录时点赞 / 收藏统一拉起登录弹窗

说明：

- 当前路线详情页不再只是展示数据，已经具备真实互动闭环
- 点赞、收藏状态会在多页面之间同步更新

### 1.2 评论区已从一次性 mock 展示升级为真实分页加载

已完成：

- `GET /api/trails/{id}/reviews` 支持分页参数
- 评论列表按“顶级评论块”分批请求
- 详情页评论区已接真实分页数据
- 底部自动加载更多评论

说明：

- 现在不是整棵评论树一次性全量返回
- 每次加载的是一批顶级评论及其完整回复树

### 1.3 评论删除能力已完成

已完成：

- `DELETE /api/reviews/{id}`
- 仅评论作者本人可删除
- 删除后详情页评论数与评分同步刷新

已完成前端体验：

- 删除入口只对本人评论显示
- 删除前使用项目内统一确认弹窗
- 删除后即时移除对应评论节点

### 1.4 评论用户卡片已完成

已完成：

- `GET /api/users/{id}/card`
- 顶级 / 二级 / 三级评论的头像与昵称都可点击
- 用户卡片弹层已接真实接口

当前卡片字段：

- `id`
- `username`
- `avatar`
- `avatarMediaUrl`
- `role`
- `joinDate`
- `postCount`
- `savedCount`
- `location`
- `bio`

说明：

- `postCount` 由路线发布数据聚合得到
- `savedCount` 由收藏关系表聚合得到
- `bio` / `location` 当前仍由后端默认值兜底

### 1.5 当前用户长整型 ID 精度问题已修复

已完成：

- `/api/auth/me` 中的长整型 ID 改为字符串序列化
- 前端当前用户类型与 store 改为字符串 ID
- 评论删除按钮显示逻辑已改为基于字符串 ID 比较

说明：

- 之前“本人评论不显示删除按钮”的根因是 19 位长整型在前端丢失精度
- 当前这条链路已经统一按字符串处理

### 1.6 删除确认弹窗已改为项目内统一组件

已完成：

- 新增通用确认弹窗组件
- 评论删除已接入统一确认弹窗
- 弹窗样式已适配当前主题色、白天模式与暗黑模式

说明：

- 当前不再使用浏览器原生 `window.confirm`
- 后续退出登录、清空草稿等二次确认动作也可以复用这套组件

## 5. 下一步最推荐的工作

最推荐优先推进的任务是：

1. 给 `users` 表补真实 `bio` / `location` 字段，并接个人资料编辑
2. 规划并落地 GPX / KML 轨迹文件上传与解析
3. 再推进天气能力迁移到后端聚合接口

推荐原因：

- 用户卡片已经上线，当前 `bio` / `location` 只是默认值兜底，体验上最值得先补齐
- GPX / KML 越早建模越好，避免后续路线发布、地图展示和数据结构返工
- 天气迁移虽然重要，但当前仍有前端直连可用，优先级低于用户资料真实化

### 建议优先级判断

最值得优先补的是：

1. 用户资料真实字段
2. GPX / KML 轨迹文件方案
3. 天气后端迁移

说明：

- 用户资料字段属于已经上线功能的缺口补齐
- GPX / KML 属于后续路线内容能力的基础建模
- 天气迁移优先级高于纯样式优化，但低于上面两项
