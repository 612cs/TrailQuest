# 当前开发进度与交接

这份文档用于记录 2026-03-20 已完成的工作、当前真实可用能力、剩余未完成项，以及下一步最推荐继续推进的方向，方便下一次开发时直接接上。

## 1. 今天已完成的内容

### 1.1 登录失败引导注册已完成

已完成：

- 登录失败时区分“用户不存在”和“密码错误”
- 后端登录失败新增稳定业务码 `USER_NOT_FOUND`
- 认证弹窗在“用户不存在”时显示“去注册”引导
- 点击后直接切到注册补充信息页
- 当前邮箱会自动带入注册流程

说明：

- 现在登录失败不再只有统一的“邮箱或密码不正确”
- 已注册用户输错密码不会被误导去注册

### 1.2 社区卡片真实分享能力已完成

已完成：

- 社区页路线卡片底部分享按钮接入真实分享能力
- 分享行为与路线详情页统一
- 优先调用 Web Share API
- 不支持系统分享时自动复制路线详情链接

说明：

- 当前社区页与详情页分享逻辑已经统一收口
- 分享目标统一为路线详情页 URL，而不是社区流页面 URL

### 1.3 个人页“我的发布 / 我的收藏”已对齐数据库

已完成：

- 个人页不再使用假数据
- `我的发布` 接真实发布路线列表
- `我的收藏` 接真实收藏路线列表
- 头部 `我的发布 / 已保存` 统计已接真实聚合数据
- 每次请求 10 条，已切为滚动加载
- 个人页列表已加入轻量虚拟列表

说明：

- 当前个人页已经不是静态 UI，而是数据库真实数据视图
- 收藏页取消收藏后，条目会立即移除，统计也会同步更新

### 1.4 个人页返回后保留 tab 状态已完成

已完成：

- 个人页当前 tab 状态已同步到路由 query
- 从详情页返回个人页时，能继续停留在原 tab
- 刷新页面后也能恢复当前 tab

说明：

- 当前没有依赖 `KeepAlive` 来记忆 tab
- 状态来源已经明确落到 URL 上，调试和分享都更稳定

### 1.5 GPX / KML 原始文件上传、解析与真实发布链路已完成

已完成：

- 发布页支持上传 `GPX / KML`
- 原始轨迹文件保存到 OSS
- `media_files` 已扩展 `trail_track`
- 新增 `trail_tracks` 轨迹解析表
- 发布路线时可提交 `trackMediaId`
- 后端会读取 OSS 原文件并解析
- 解析结果写入结构化轨迹数据
- 距离 / 爬升 / 时长支持自动回填
- 路线详情已返回轨迹对象，包含：
  - `downloadUrl`
  - `geoJson`
  - `distanceMeters`
  - `elevationGainMeters`
  - `bounds`
  - `startPoint`
  - `endPoint`

说明：

- 当前已经形成“原文件保留 + 结构化结果入库”的双层设计
- 这套结构已经兼容后续“下载轨迹”和 “Three.js 渲染”

### 1.6 雪花 ID 精度问题已统一修复

已完成：

- 后端长整型 ID 统一按字符串序列化
- 前端发布链路、轨迹链路、详情链路统一按字符串传递 ID
- 修复了因为 JS Number 精度丢失导致的 `INVALID_MEDIA_FILE`

说明：

- 之前“OSS 里有文件，但发布时说无权使用”的根因就是雪花 ID 丢精度
- 当前媒体文件、轨迹文件、路线 ID 这条链路都已统一按字符串处理

### 1.7 KML `gx:Track` 解析已补齐

已完成：

说明：

### 1.8 Three.js 轨迹查看器已正式接入发布页与详情页

已完成：

- 将根目录 `trails.html` 的核心 Three.js 场景抽离为正式 Vue 组件

- 路线详情页顶部新增独立一屏的 Three.js 轨迹查看器

说明：

### 1.9 Three.js 查看器路牌、最高点、小鸟与天气场景已补齐

已完成：

说明：

- 路牌、最高点、鸟群和天气都已经接入统一的 Three.js 生命周期

## 5. 下一步最推荐的工作

最推荐优先推进的任务是：

1. 打磨 Three.js 查看器观感与交互细节
2. 在详情页显式补“下载轨迹”按钮入口
3. 继续规划 Three.js 轨迹渲染的更强表现，例如动画镜头、分段里程事件、路线节奏演出

推荐原因：

- 当前 Three.js 主链路已经跑通，最适合先做一轮质感打磨
- 下载轨迹的底层数据与接口已经具备，补入口成本低、用户感知强
- 更强的 Three.js 表现层能力现在已经有了稳定底座，后续扩展不会再推倒重来

### 建议优先级判断

最值得优先补的是：

1. Three.js 体验打磨
2. 下载轨迹入口
3. Three.js 更强表现层能力

说明：

- 当前不缺“能不能用”，而是进入“体验够不够好”的阶段
- 继续在 Three.js 上做体验增强，收益会高于去补零散小功能
